                    # response

                    表示服务器返回的数据，可能是任意数据类型，由responseType决定

                    # readyState

                    表示实例对象当前状态，该属性为只读

                    值描述
                    0实例已生成，但未调用 open()
                    1open() 已调用，但未调用 send() ,但可以设置请求头
                    2send() 已调用，且服务器返回的头信息与状态码已收到
                    3正在接收服务器传来的数据体。如果`responseType:text
                    4服务器返回数据已完全接收或本次请求失败

                    # responseType

                    表述服务器返回的数据类型，open() ~ send() 之前设置

                    值描述
                    ""默认，文本
                    text文本
                    jsonJSON对象
                    blob二进制数据Blob对象
                    arraybuffer二进制数据ArrayBuffer
                    documentDocument，文档对象

                    # responseText

                    表示从服务器接收到的字符串，只有请求完成接收后才包含完整数据

                    # responseXML

                    表示从服务器接收到的 HTML 或 XML 文档对象，该属性只读。如果请求失败，或收到的数据无法解析为 XML 或 HTML，该属性值为null。

                    该属性生效的前提

                    • Content-Type头信息等于text/xml或application/xml

                    • responseType属性要设为document

                    如果 HTTP 回应的Content-Type头信息不等于text/xml和application/xml，但是想从responseXML拿到数据（即把数据按照 DOM 格式解析），那么需要手动调用XMLHttpRequest.overrideMimeType()方法，强制进行 XML 解析

                    该属性得到的数据，是直接解析后的文档 DOM 树。

                    # responseURL

                    表示发送数据的服务器地址，，这个属性的值与open()方法指定的请求网址不一定相同。如果服务器端发生跳转，这个属性返回最后实际返回数据的网址。另外，如果原始 URL 包括锚点（fragment），该属性会把锚点剥离

                    # withCredentials

                    表示跨域请求时，用户信息（比如 Cookie 和认证的 HTTP 头信息）是否会包含在请求之中，默认为false

                    # setRequestHeader

                    设置请求头信息

                    • Content-Type 发送数据的格式
                    • Content-Length 发送数据的长度
                    Content-Type描述
                    text/htmlHTML
                    text/plain纯文本
                    text/xmlXML
                    image/gifgif图片
                    image/jpegjpeg图片
                    image/pngpng图片
                    application/xmlXML
                    application/pdfPDF
                    application/jsonJSON字符串格式
                    application/mswordWord
                    application/octet-stream二进制流
                    application/x-www-form-urlencodedkey1=value1&key2=value2的格式发送，特殊字符需要转义
                    multipart/form-data提交表单
                    xhr.setRequestHeader('Content-Type','application/x-www-form-urlencoded')

                    # overrideMimeType

                    指定 MIME 类型，覆盖服务器返回的类型，从而让浏览器进行不一样的处理。

                    如服务器返回的数据类型text/xml，但浏览器解析失败，无法获取数据。为获取原始数据，可以将IME 类型改成text/plain，此时浏览器不会自动解析，从而可以获取原始文本

                    xhr.overrideMinmeType('text/plain') //send()之前调用

                    仅在服务器无法返回某种数据类型时使用

                    # Get请求

                    document.querySelector('input').onblur = function() {
	//创建异步对象
	var xhr = new XMLHttpRequest();
	
	//设置请求行(get请求，数据写在url后面)
	xhr.open('get','xxx.php?name='+this.value);
	
	//设置请求头(get请求头可以省略，post不发数据可以省略)
	
	//回调函数1
	xhr.onload = function() {}
	
	//请求主体(get-空或null，post不发数据-空/null)
	xhr.send()
}

                    # Post请求

                    document.querySelector('input').onblur = function() {
	//创建异步对象
	var xhr = new XMLHttpRequest();
	
	//设置请求行,第三个参数true(异步) false(同步)
	xhr.open('post','xxx.php');
	
	//设置请求头
	xhr.setRquestHeader('content-type','application/x-www-form-urlencoded')
	
	//回调函数2
	xhr.onreadystatechange = function() {
	//判断状态 请求是否成功
		if(xhr.readyState==4 && xhr.status==200){
			xhr.responseText
		}
	}
	
	//请求主体(post数据写在send,多个参数&连接)
	xhr.send('name='+this.name&'pwd='+this.pwd)
}

                    function get(url,data,success) {
	var xhr = new XMLHttpRequest();
	
	if(data){
		url+='?';
		url+=data;
	}
	xhr.open('get',url);

	xhr.onreadystatechange = function() {

	if(xhr.redyState==4 && xhr.status==200){
			success(xhr.responseText);
			console.log(xhr.responseText);

		}

	xhr.send()

                    # post请求封装

                    function post(url,data,success) {
	var xhr = new XMLHttpRequest();
	xhr.open('post',url);

	if(data){
		xhr.setRquestHeader('content-type','application/x-www-form-urlencoded')
	}
	
	xhr.onreadystatechange = function() {
	if(xhr.redyState==4 && xhr.status==200){
			success(xhr.responseText)
		}
	}
	
	xhr.send(data)
}

function ajax(option){
	var {url,type,data,success} = option;
	var xhr = new XMLHttpRequest();
	xhr.open(type,url);

	if(type='get'&data){
		url+='?'+data;
		data=null;
	}
	if(type='post'&data){
		xhr.setRquestHeader('content-type','application/x-www-form-urlencoded')
	}
	
	xhr.onreadystatechange = function() {
	if(xhr.redyState==4 && xhr.status==200){
			var content-type = xhr.getResponseHeader('Content-Type');
			if(content-type.indexOf('json')!=-1){
				success(JSON.parse(xhr.responseText));
			}
			else if(content-type.indexOf('xml')!=-1){
				success(xhr.responseXML);
			}
			else{
				success(xhr.responseText);
			}
		}
	}
	
	xhr.send(data)
}

                    let AjaxPromise = function (param) {
  return new Promise(function (resolve, reject) {
    let xhr = new XMLHttpRequest();
    xhr.open("get", param.url, true);//post同理
    xhr.onload = resolve;
    xhr.onerror = reject;
    xhr.send();
  })
}

                    # Jquery Ajax

                    全局ajax事件，每当一个ajax事件达成条件就会触发，绑定对象$(document)或dom元素

                    方法（执行顺序）描述
                    .ajaxStart()Ajax请求开始，xhr创建，open前
                    beforeSend
                    .ajaxSend()Ajax请求即将发送，send前
                    success
                    .ajaxSuccess()Ajax请求成功完成
                    erroe
                    .ajaxError()Ajax请求出错时
                    complete
                    .ajaxComplete()Ajax请求完成，单个
                    .ajaxStop()Ajax请求完成，所有

                    # 基本语法

                    $.ajax{
	url:请求地址
	type:请求方式,默认get
	async:是否异步，默认true
	data:发送到服务器的数据,支持对象{name:'xxx'}
	dataType:预期服务器返回的类型
	contentType:设置请求头,能够自动转化数据
	success:请求成功回调函数
	error:请求失败回调函数(XMLHttpRequest,textStatus,errorThrown)[参数1:异步对象  参数2:错误信息  参数3:错误位置 ]
	complete:请求完成后触发
}
