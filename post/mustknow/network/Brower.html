                              # 浏览器内核

                              渲染引擎，负责对网页语法的解释并渲染网页

                              Chrome内核：webkeit--》Blink

                              # DOCTYPE作用

                              声明文件类型定义，告诉浏览器应该按那个规范解析文档

                              严格模式：按W3C标准解析

                              混合模式：兼容模式，按浏览器自己的方式解析

                              # 页面三层构成

                              结构层：HTML，实现页面结构

                              表示层：CSS，完成页面的表现和风格

                              行为层：Javascript实现客户端的交互功能

                              # Web标准与W3C

                              web标准：将三部分独立分开，使其更具有模块化

                              W3C：对web标准提出规范化要求

                              • html，标签字母小写，标签闭合，不允许随意嵌套
                              • js、css：尽量少用行内样式，尽量使用外链样式和脚本

                              # URL到页面渲染

                              • 域名解析，浏览器根据DNS服务器得到域名IP地址
                              • 发起TCP三次握手
                              • 建立TCP连接后向服务器发起HTTP请求
                              • 服务器接处理HTTP请求，返回HTML代码
                              • 浏览器解析HTML代码，渲染页面

                              # 回流重绘

                              首先了解浏览器得渲染过程，渲染引擎会解析HTMl文档构建DOM树，与此同时渲染引擎使用CSS解析器解析CSS文档构建CSSOM树。接下来将两棵树关联起来形成渲染树。然后浏览器按照渲染树进行布局（回流）计算得到节点得几何信息（位置，大小），根据渲染树以及回流得到的几何信息得到节点得绝对像素（重绘），最后将像素发给GPU，展示在页面上

                              何时会发生回流

                              • 添加、删除可见DOM元素
                              • 元素位置发生变化
                              • 元素尺寸发生变化
                              • 内容发生变化
                              • 浏览器窗口尺寸发生变化
                              • 页面首次渲染时

                              回流一定会引起重绘，重绘不一定引起回流

                              何时发生重绘

                              • 改变背景颜色
                              • 改变文字颜色
                              • 改变边框颜色

                              减少回流重绘方法

                              • visibility代替display:none,因为前者只会触发一次重绘，后者触发回流
                              • 避免使用表格布局
                              • 避免频繁操作DOM，操作DOM时拷贝原始节点，使其脱离文档流（display:none）再进行修改(display:block)，最后替换原始元素
                              • 避免频繁读取引起回流重绘的属性（类似offsetWidth），可以将其用变量保存起来
                              • 现代浏览器大多都是通过队列机制来批量更新布局，浏览器会把修改操作放在队列中，至少一个浏览器刷新（即16.6ms）才会清空队列，但当你获取布局信息的时候，队列中可能有会影响这些属性或方法返回值的操作，即使没有，浏览器也会强制清空队列，触发回流与重绘来确保返回正确的值。

                              # 同源跨域

                              原理

                              跨域是指浏览器不能执行其他网站的脚本。它是浏览器同源策略造成的，是浏览器对JS实施的安全限制。协议、端口、域名、任何一个不同则为不同源，不同源之间无法访问。

                              解决

                              • JSONP：动态创建script，请求带参网址实现跨域，缺点是只支持get方法，需要后端配合返回指定格式的数据
                              • CORS：服务端设置Access-Control-Allow-Origin
                              • 代理跨域：启动代理服务器
                              一.cross(html5支持)
后台设置 header('Access-Control-Allow-Origin:*');

//有时候也需要添加以下设置
header("Access-Control-Allow-Headers: Origin, X-Requested-With, Content-Type, Accept");
header('Access-Control-Allow-Methods: GET, POST, PUT');

二、.JSNOP
1. <script src="xxxx.xxx?callback=callbackname"></script>
原理：利用script的src属性支持跨域访问
	  写上需要请求的页面，拼接方法名，发送服务器
	  服务器将数据作为方法名的参数(字符串拼接)，返回浏览器
	  浏览器将返回的方法名(data)作为js解析
	  
2. dataType:JSONP,会自动在请求url后拼接?'callback=xxx',在生成的回调函数中将数据返回，供success使用

                              # 浏览器存储

                              • 请求携带
                                • cookie数据始终在同源http请求中携带
                                • WebStorage不会自动将数据发送给服务器
                              • 存储大小
                                • cookie：4k；webstorage：5M
                              • 有效时间
                                • localStorage，永久存储鱼浏览器除非手动删除
                                • sessionStorage，数据在当前浏览器窗口关闭之后自动删除
                                • cookie在过期时间之内有效

                              • 渐进增强：针对低版本浏览器进⾏构建⻚⾯，保证最基本的功能，然后再针对⾼级浏览器 进⾏效果、交互等改进和追加功能达到更好的⽤户体验。
