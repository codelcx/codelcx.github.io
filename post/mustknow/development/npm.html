                # main

                "main" : "dist/vue.runtime.common.js"

                const vue = require('vue') 自动寻找module字段对应的node_modules/vue/dist/vue.runtime.esm.js

                此方式为CJS引入方式

                # module

                "module": "dist/vue.runtime.esm.js"

                import vue from 'vue' 自动寻找main字段对应的 node_modules/vue/dist/vue.runtime.common.js

                此方式为ESM引入方式

                通常Web项目是以ESM方式引入包，它的查找方式为 module -> main -> index.js

                # browser

                针对浏览器化境下的入口文件

                "browser": "./lib/browser/main.js"

                它将替换main字段作为浏览器环境下的入口文件

                # exports

                它的优先级大于[main、modules、browser]

                路径封装

                // 该包仅存在默认导出
"exports": "./index.js" 

// 等价写法
"exports": {		 
    "."； "./index.js"
}

                定义该字段后，仅支持引入包自身，禁止引入其它子路径

                // correct
import Test from 'test'

// error
import Test from 'test/example.js'

                当然也可以定义子路径的导出

                "exports": {
    ".": './index.js'
    "./example.js": "./src/example.js"
}

                条件导出

                位置越靠前优先级越高

                "exports": {
    // ESM
    "import": "./index-module.js",
    // CJS
    "require": "./index-require.cjs"
    // ESM / CJS
    "default": "./index-default.js"
}

                嵌套条件

                支持不同引入方式进行不同条件的导出

                支持不同运行环境中不同引入方式进行不同条件的导出

                "exports": {
    "node": {
        "import": "./import-node.mjs",
        "require": "./require-node.cjs",
    },
    "default": "./default.js"
}
