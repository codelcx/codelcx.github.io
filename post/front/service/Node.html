                              # 模块开发

                              # 模块规范

                              Node遵循CommonJS模块规范，规定模块特性与各个模块之间如何相互依赖

                              • 每个模块内部，module变量代表当前模块
                              • module变量是一个对象，它的exports属性是对外接口
                              • 加载模块时，其实是记载module.exports属性

                              # 导入导出

                              export 是 module.export 的别名，默认情况下指向同一个对象，后者优先级最高，导入时永远是module.export所指向的对象

                              //main.js export导出
function foo(){}
export.foo = foo

//index.js 导入
const a = require('./main.js') //后缀名可以省略

//main.js module.export导出s
module.exports.foo = foo

                              //main.js 两者的区别
export = { name:'xxx'}
module.exports = { name:'yyy'}

//导入
const name = require('./main')
console.log(name) //{name:'yyy'},以module.export对象为主

                              # 加载机制

                              • 优先缓存中加载，第一次加载会添加至缓存
                              • 内置模块加载优先级最高
                              • 自定义模块需要指定``./ 或 ../`,否则加载内置或第三方模块

                              # 查找规则

                              //require模块查找规则(无后缀) ./find
1.完整路径，直接引入
2.模块后缀省略，先找同名js文件再找同名文件夹
3.若找到同名文件夹，找其中的index.js
4.若文件中没有index.js则去当前文件夹中的package.json中查找main选项中的入口文件
5.若没有指定入口文件，或没指定入口文件则报错，模块查找失败

//require模块查找规则(无路径无后缀) find
1.假设是系统模块
2.查找node_modules文件夹
3.是否存在该名字的js文件
4.是否存在该名字的文件夹
5.文件夹下是否存在index.js
6.无index.js查找该文件夹中的package.json中main选项确定的模块入口文件
7.否则查找失败

                              # 版本规范

                              包版本以点分十进制形式定义，如1.10.0

                              第一位：大版本，从底层修改

                              第二位：功能版本，底层未修改，对功能进行修改或添加

                              第三位：BUG修复版本

                              ~1.2.3 版本号<1.3.0
^1.2.3 版本号<2

                              fs.readFIle(path,[option],callback)

fs.readFile('./new1.txt', 'utf-8', (err, data) => {

  }

//成功，err为null
//失败，err为错误对象，data为undefined

                              3、动态路径

                              使用fs模块时，如果提供相对路径，容易出现路径动态拼接错误。因为代码运行时回执行node命令所处的目录，动态拼接文件的完整路径。

                              //文件路径：C:\Users\Admin\Desktop\Node\new.txt

C:\Users\Admin\Desktop\Node> node new.js //执行成功
C:\Users\Admin\Desktop> node .\Node\new.js //执行失败，错误路径C:\Users\Admin\Desktop\new.txt

                              解决方案：使用__dirname,表示当前文件所处目录

                              __dirname+'/new.txt' //修改文件路径，一定不能缺少\，也不能加.
C:\Users\Admin\Desktop> node .\Node\new.js //执行成功

                              # 路径模块

                              提供一系列方法和属性，满足用户对路径处理的需求

                              方法描述
                              path.join将多个路径片段拼接
                              path.basename从路径中，解析出文件名
                              path.extname从路径中，获取文件扩展名
                              const path = require('path')

//1.path.join
const pathStr1 = path.join('/a','/b/c','../','d') //\a\b\d
const pathStr2 = path.join(__dirname,'/new.txt') //减少+的使用,可以使用./new.txt而不报错

//2.path.basename
const pathStr3 = path.basename('Node/index.html') //index.html
const pathStr4 = path.basename('Node/index.html','html')//index 

//3.path.extname
const pathStr5 = path.extname('Node/index.html')//html

                              # HTTP模块

                              通过该模块可以将普通电脑转换为Web服务器，从而对外提供资源服务

                              //创建最基本的服务器步骤
//1.导入http模块
const http = require('http')
//2.创建web服务器
const server = http.createServer()
//3.为示例绑定request事件，监听客户端请求
server.on('request', (req, res) => {})
//4.启动服务器，指明端口号
server.listen(8080, () => {})

                              1、请求对象

                              req:请求对象，包含客户端相关数据和属性

                              • eq.url，请求地址，不包括ip地址，/index.html
                              • req.method，请求方法

                              2、响应对象

                              res:响应对象，包好服务器相关数据与属性

                              • res.end() 向客户端发送指定内容并结束此次请求
                              server.on('request',(req,res)=>{
    //中文乱码问题
    const str = "中文乱码问题"
  res.setHeader('Content-Type', 'text/html; charset=utf-8');
  res.end(str)
})

                              3、具体案例

                              const http = require('http')
const path = require('path')
const fs = require('fs')

const server = http.createServer()

server.on('request', (req, res) => {
  res.setHeader('Content-Type', 'text/html; charset=utf-8');
  const fpath = path.join(__dirname, req.url);
  console.log(fpath);
  fs.readFile(fpath, (err, data) => {
    if (err) return res.end('404')
    res.end(data)
  })
})

server.listen(80, () => {})

                              # 文件分析

                              # packpage.json

                              项目描述文件，记录当前项目信息，例如项目名称，版本（范围），依赖的第三方模块..

                              当他人使用项目时并不需要为其传递node_modules文件夹，因为可以根据package.json下载依赖

                              npm init //初始化项目，生成package.json
npm init -y //使用默认值，文件夹不允许是中文

npm install //根据package.json 下载全部依赖
npm install --production //只会下载项目运行依赖

npm i packageName --save-dev //只在开发阶段使用

npm i packageName -g //全局安装

                              # package-lock.json

                              锁定包的具体版本，确保再次下载时不会因为包的版本为产生问题

                              加快下载速度，因为该文件记录项目所依赖第三方包的树状结构和下载地址，重新安装只需要下载无需额外工作
